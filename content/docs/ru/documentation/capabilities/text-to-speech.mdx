DVoice предлагает мощные возможности преобразования текста в речь (TTS), позволяя преобразовывать текст в реалистичное аудио с использованием двух различных конечных точек. Эти конечные точки подходят для разных случаев использования, обеспечивая гибкость как для генерации отдельных аудиофайлов, так и для потоковой передачи в реальном времени.

## Доступные конечные точки

DVoice поддерживает следующие конечные точки TTS:

- **Единичная конечная точка**: Преобразует текст в аудио в одном HTTP-запросе POST, идеально подходит для создания полных аудиофайлов. Доступна по адресу `https://oyqiz.airi.uz/api/v2/tts` (см. [как использовать](../../api/tts/single) в справочнике API).
- **Потоковая конечная точка**: Передает аудио в реальном времени через WebSocket, подходит для приложений с низкой задержкой. Подключение доступно по адресу `wss://oyqiz.airi.uz/stream` (см. [как использовать](../../api/tts/stream) в справочнике API).

## Обязательные поля

Обе конечные точки TTS требуют следующих полей:

- **text**: Строка с текстом, который нужно преобразовать в речь.
- **model**: Строка, указывающая модель голоса (например, "Iroda" или "Surayyo_v2").
- **format**: Строка, определяющая аудиоформат, поддерживающие следующие варианты:
  - `MP3`
  - `WAV`
  - `AAC`
  - `OGG`
  - `FLAC`

## Опциональные поля

- **data**: Объект для дополнительных параметров, поддерживаемых конкретными моделями (при необходимости). Это поле опционально и зависит от выбранной модели.

## Поддерживаемые модели

DVoice предоставляет две модели TTS, каждая из которых адаптирована для узбекского языка с уникальными особенностями:

- **Iroda**: Универсальная модель, разработанная исключительно для узбекского языка. Не поддерживает дополнительных полей в объекте `data`.
- **Surayyo_v2**: Усовершенствованная модель, также ограниченная узбекским языком, с поддержкой эмоциональных тонов голоса. Включает опциональное поле `emo` в объекте `data`, которое может быть установлено в:
  - `Happy` (Счастливый)
  - `Strict` (Строгий)
  - `Neural` (Нейтральный)

## Примечания по использованию

- Максимальная длина текста: 5,000 символов, хотя это может измениться.
- Поддерживаемые аудиоформаты включают `MP3`, `WAV`, `OGG`, `AAC` и `FLAC`.
- Тарифы рассчитываются на основе длины обработанного текста.
- Для потоковой конечной точки обеспечьте стабильное WebSocket-соединение для непрерывной передачи аудио.
- Ознакомьтесь со [справочником API](../../api/tts/single) для получения подробной информации об использовании API, обработке ошибок и дополнительных технических деталях.
