DVoice применяет ограничения скорости, чтобы обеспечить справедливое использование и поддерживать стабильность сервиса для всех пользователей. Следующие ограничения действуют для конечных точек TTS (`https://oyqiz.airi.uz/api/v2/tts`, `ws://oyqiz.airi.uz/stream`) и STT (`/stt`).

## Текущие ограничения скорости

### Преобразование текста в речь (TTS)

- **Запросов в минуту**: 60 запросов в минуту на один API-токен.

Это ограничение применяется к общему количеству запросов для одиночных и потоковых преобразований TTS, отслеживаемых в течение скользящего окна в одну минуту.

### Преобразование речи в текст (STT)

- **Длительность аудио в минуту**: 5 минут (300 секунд) аудио в минуту на один API-токен.

Это ограничение применяется к общей длительности аудио, обработанного для транскрипции STT, отслеживаемой в течение скользящего окна в одну минуту.

## Как применяются ограничения

- Ограничения отслеживаются для каждого API-токена и сбрасываются каждую минуту.
- Превышение лимита вернёт HTTP-статус `429 Too Many Requests` с заголовком `Retry-After`, указывающим, когда можно повторить попытку.
- Для потоковой передачи через WebSocket (TTS) или обработки STT превышение лимита приведёт к закрытию соединения или отклонению запроса с сообщением об ошибке.

## Будущие планы

Более высокие лимиты могут быть доступны для платных подписок в будущем. Ожидаемые увеличения могут включать 120 запросов в минуту для TTS и 10 минут в минуту для STT. Подробности будут обновлены здесь по мере их появления.

## Обновления ограничений скорости

Ограничения скорости могут быть скорректированы в зависимости от шаблонов использования и производительности сервиса. Обновления будут сообщаться через панель управления DVoice и эту документацию, с последней информацией, отражающей текущую дату. Ваше дальнейшее использование API после изменений указывает на принятие новых лимитов.
