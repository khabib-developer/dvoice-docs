# Аутентификация

API DVoice использует аутентификацию на основе токенов. Валидный API-токен должен быть включен во все запросы к конечным точкам Text to Speech и Speech to Text.

## Получение токена

1. Перейдите на [voice.airi.uz](https://voice.airi.uz).
2. Войдите в систему, используя учетные данные вашей учетной записи DVoice.
3. Перейдите в раздел API, чтобы создать или скопировать ваш API-токен.
4. Храните токен в безопасности.

<Callout type="warning">
  Не делитесь своим API-токеном публично. Если он скомпрометирован, сгенерируйте
  новый на [voice.airi.uz](https://voice.airi.uz).
</Callout>

## Выполнение аутентифицированных запросов

### Единичная конечная точка (HTTP)

Добавьте токен в заголовок `token` вашего HTTP-запроса POST.

**Пример заголовка**:

```
token: <your-token>
```

### Потоковая конечная точка (WebSocket)

Включите токен в заголовок `token` при подключении к URL-адресу потока.

**Пример подключения** (сырой WebSocket):

```
token: <your-token>
```

При использовании SDK настройте токен при инициализации клиента (см. [Использование SDK](../developer-guide/sdk-libraries)).

## Обработка ошибок аутентификации

| Статус/Код | Сообщение                          | Описание                        |
| ---------- | ---------------------------------- | ------------------------------- |
| 401        | Неверный токен                     | Токен недействителен или истек. |
| 403        | Для аутентификации требуется токен | Токен не был предоставлен.      |

Для подключений WebSocket недействительный или отсутствующий токен вызывает ответ `401 Unauthorized` или `403 Forbidden`, после чего соединение разрывается.

## Дополнительные заметки

- Токены содержат идентификатор пользователя и имя токена, которые проверяются на стороне сервера.
- Если вы получили ошибку `401 Invalid Token`, проверьте валидность токена на [voice.airi.uz](https://voice.airi.uz).

<Callout type="info">
  Обратитесь в поддержку через [voice.airi.uz](https://voice.airi.uz) для
  решения проблем с генерацией токена.
</Callout>
